===================================
CONCEPTOS TÉCNICOS DEL PROYECTO
===================================

1. MVVM (Model-View-ViewModel)
================================
MVVM es un patrón de arquitectura de software que separa la lógica de presentación de la interfaz de usuario.

Componentes:
- Model: Representa los datos y la lógica de negocio (en este proyecto: Product, ProductRepository)
- View: La interfaz de usuario (en este proyecto: Componentes de Jetpack Compose)
- ViewModel: Actúa como intermediario entre Model y View, maneja el estado de la UI y la lógica de presentación

Beneficios:
- Separación de responsabilidades
- Facilita las pruebas unitarias
- Permite que la UI sea reactiva a los cambios de datos
- Mejora la mantenibilidad del código

En este proyecto:
- ProductViewModel: Maneja el estado y las operaciones CRUD
- ProductRepository: Abstrae el acceso a datos de Firebase
- Composables: Representan la vista (UI)


2. FIREBASE AUTHENTICATION
===========================
Firebase Authentication proporciona servicios backend, SDKs y bibliotecas de UI para autenticar usuarios en la aplicación.

Características:
- Múltiples métodos de autenticación (email/password, Google, Facebook, etc.)
- Gestión segura de credenciales
- Tokens de sesión automáticos
- Integración con otros servicios de Firebase

En este proyecto:
- Autenticación por email y password
- FirebaseAuth instance para gestionar usuarios
- Estados de autenticación (logged in/logged out)
- Registro y login de usuarios


3. FIRESTORE (Cloud Firestore)
===============================
Cloud Firestore es una base de datos NoSQL flexible y escalable en la nube de Firebase.

Características:
- Base de datos en tiempo real
- Sincronización offline
- Escalabilidad automática
- Consultas potentes
- Estructura de documentos y colecciones

Estructura de datos:
- Colecciones: Contenedores de documentos (ej: "products")
- Documentos: Unidades básicas de almacenamiento (ej: un producto específico)
- Campos: Datos dentro de los documentos (ej: name, price, description)

En este proyecto:
- Colección "products" para almacenar los productos
- Cada documento representa un producto individual
- Operaciones CRUD en tiempo real


4. JETPACK COMPOSE
===================
Jetpack Compose es el kit de herramientas moderno de Android para crear interfaces de usuario nativas.

Características principales:
- UI declarativa: Describes cómo debería verse la UI
- Menos código: Más simple y fácil de mantener
- Desarrollo intuitivo: API intuitiva y fácil de usar
- Compatibilidad con Kotlin: Totalmente integrado con Kotlin

Conceptos clave:
- Composables: Funciones anotadas con @Composable que definen la UI
- Recomposición: La UI se actualiza automáticamente cuando cambia el estado
- State hoisting: Elevación del estado para hacerlo accesible
- Material Design 3: Componentes modernos de diseño

En este proyecto:
- Todas las pantallas son Composables
- Uso de Material Design 3 (Material3)
- Navigation Compose para navegación entre pantallas
- Estado reactivo con ViewModel


5. CRUD
=======
CRUD es el acrónimo de las cuatro operaciones básicas en bases de datos:

CREATE (Crear):
- Agregar nuevos productos a Firestore
- Función: addProduct() en ProductViewModel
- UI: ProductFormScreen con campos de entrada

READ (Leer):
- Obtener y mostrar lista de productos
- Función: getProducts() en ProductViewModel
- UI: ProductListScreen con LazyColumn

UPDATE (Actualizar):
- Modificar productos existentes
- Función: updateProduct() en ProductViewModel
- UI: ProductFormScreen en modo edición

DELETE (Eliminar):
- Borrar productos de la base de datos
- Función: deleteProduct() en ProductViewModel
- UI: Botón de eliminar en cada producto


FLUJO DE DATOS EN LA APLICACIÓN
================================

1. Usuario interactúa con la UI (Composables)
   ↓
2. La UI llama a funciones del ViewModel
   ↓
3. El ViewModel ejecuta lógica de negocio y llama al Repository
   ↓
4. El Repository interactúa con Firebase (Auth o Firestore)
   ↓
5. Firebase procesa la operación y responde
   ↓
6. El Repository devuelve el resultado al ViewModel
   ↓
7. El ViewModel actualiza el estado (StateFlow)
   ↓
8. La UI se recompone automáticamente mostrando los nuevos datos
